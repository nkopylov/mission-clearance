Mission Clearance

Security harness for Claude Code. Intercepts every tool call, evaluates it against capability-scoped policies, and blocks dangerous operations before they reach the outside world. For multi-agent deployments, adds identity-aware delegation verification with RBAC, chain verification, and trust-level enforcement.

How It Works

Think of it as an OS kernel for your AI agent: missions are processes, capabilities are file descriptors, and policies are syscall filters. Every external action is checked, classified, and logged in a tamper-evident trace. 

Claude Code ──tool call──> PreToolUse Hook ──> MC Server
                                                  │
                                          Classify + Evaluate
                                                  │
                              ┌─────────┬─────────┼─────────┐
                          Allow      Deny     Escalate
                              │         │         │
                           proceed    block    Claude judges
                                               (via MCP tools)
1

Intercept

Claude Code attempts a tool call (Bash, Edit, Write, etc.). The PreToolUse hook intercepts it, expands context (script contents, package.json lifecycle hooks, cross-file function bodies), and sends the enriched operation to the MC server.

2

Classify

The kernel classifies the operation: resource type, destructiveness, data flow direction, pattern matching against known attack vectors.

3

Evaluate

The policy pipeline evaluates deterministic rules and returns one of three verdicts: Allow, Deny, or Escalate (where Claude itself acts as the LLM judge).

4

Record

Every decision is recorded in a tamper-evident trace with SHA-256 hash chaining. Anomalies like taint propagation, goal drift, and prompt injection are tracked automatically.

Getting Started

1

Build

Clone the repository and build the Rust binary:

cd /path/to/mission-clearance
cargo build --release
2

Install the plugin

The plugin lives at ~/.claude/plugins/local/mission-clearance/ and includes the hook script, MCP server, and slash commands. Ensure the mission-clearance binary is on your PATH.

~/.claude/plugins/local/mission-clearance/
  .claude-plugin/plugin.json    # Plugin manifest
  .mcp.json                     # MCP server configuration
  hooks/
    hooks.json                  # Hook registration
    pre-tool-use.py             # PreToolUse hook script
  commands/
    start.md, stop.md, ...      # Slash commands
  scripts/
    mc-mcp-server.py            # MCP server (stdio JSON-RPC)
3

Start a session

Launch the MC server, create a root mission, and activate gating:

/mission-clearance:start "implement user authentication"
4

Work normally

All tool calls are now checked automatically. You do not need to do anything differently. The hook runs transparently on every tool call.

Permission Graph

For multi-agent and enterprise deployments, Mission Clearance adds a permission graph that tracks who authorized each agent and whether they had the authority to do so.

1

Identity

Resolve who is making the request via OIDC/JWT, API keys, or mission tokens. Map external identities to principals in the graph.

2

RBAC

Check what the principal can do via role assignments with hierarchical inheritance, scoped permissions, and separation of duties.

3

Chain Verification

Walk the delegation chain from leaf agent to human root, verifying every link: expiration, scope, sub-delegation rights, and principal trust levels.

4

Trust Enforcement

Context-aware rules enforce trust levels: deny destructive ops from low-trust agents, escalate chains without human roots, flag anomalous delegation patterns.
