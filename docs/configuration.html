Configuration

TOML config reference, environment variables, and default policies.

Configuration File

Mission Clearance uses a TOML config file (mission-clearance.toml or config/default.toml). All settings have sensible defaults. Override individual values via a local config file or environment variables.

Kernel

[kernel]

# Maximum depth of mission delegation chains
max_delegation_depth = 10

# Default timeout for missions (after which they auto-revoke)
default_mission_timeout = "2h"

# If true, operations that cannot be evaluated (e.g., server error)
# are denied rather than allowed. Recommended for production.
fail_closed = true
KeyTypeDefaultDescription
max_delegation_depth integer 10 Maximum depth of nested sub-mission delegations
default_mission_timeout duration "2h" Auto-revoke missions after this duration
fail_closed boolean true Deny operations when evaluation fails (server error, timeout)

API Server

[api]

# Address the internal HTTP server listens on
listen = "127.0.0.1:9090"
KeyTypeDefaultDescription
listen string "127.0.0.1:9090" Bind address for the internal HTTP API server

Vault

[vault]

# Path to the encrypted vault database
path = "~/.mission-clearance/vault.db"

# Key derivation source: "passphrase" uses Argon2id
master_key_source = "passphrase"
KeyTypeDefaultDescription
path string "~/.mission-clearance/vault.db" File path for the encrypted credential store
master_key_source string "passphrase" How the master key is derived ("passphrase" = Argon2id from MC_VAULT_PASSPHRASE)

Adapters

[adapters.*]

# HTTP adapter — intercepts outbound HTTP requests
[adapters.http]
listen = "127.0.0.1:8080"
tls_inspection = true
ca_cert_path = "~/.mission-clearance/ca.pem"

# Database adapter — intercepts database connections
[adapters.db]
listen = "127.0.0.1:5433"
upstream_protocol = "postgres"

# Shell adapter
[adapters.shell]
sandbox = "none"

# Tool call adapter
[adapters.tool_call]
mode = "wrapper"

Policy Pipeline

[policy_pipeline]

# The deterministic layer runs in-process. When it returns "Escalate",
# the integration layer (e.g., Claude Code hook) asks the host LLM to
# judge the operation — no separate API key or HTTP call is needed.
layers = ["deterministic"]
KeyTypeDefaultDescription
layers array ["deterministic"] Policy evaluation layers. Escalation to LLM is handled by the integration layer (hook), not the pipeline itself.

Trace

[trace]

# Path to the trace database
path = "~/.mission-clearance/trace.db"

# Verify SHA-256 chain integrity on reads
chain_integrity = true

# Export format for the mission delegation graph
graph_export_format = "dot"
KeyTypeDefaultDescription
path string "~/.mission-clearance/trace.db" File path for the append-only trace log
chain_integrity boolean true Verify SHA-256 hash chain integrity on reads
graph_export_format string "dot" Format for delegation graph export (DOT/Graphviz)

Operation Policies

[operation_policies]

# Include the built-in default policies
include = ["defaults"]

Environment Variables

VariablePurposeRequired
MC_VAULT_PASSPHRASE Passphrase for the AES-256-GCM encrypted vault. Used with Argon2id key derivation to generate the master encryption key. Only if using the vault
MC_API_KEY API key for server authentication. If not set, the server runs in open dev mode (no authentication required). No (defaults to open)

Default Policies

Built-in deterministic rules are defined in config/default-policies.toml. These policies are evaluated in priority order (highest first) and form the baseline security guarantees. Higher priority rules are evaluated first.

⛔ no-catastrophic-destruction

Deny operations classified as catastrophic and irreversible.

deny_when = "destructiveness == Catastrophic AND reversibility == Irreversible"
scope     = "global"
priority  = 100

🚫 no-self-modification

Deny operations targeting the permission system itself.

deny_when = "resource contains 'mission-clearance' OR resource contains 'system-prompt'"
scope     = "global"
priority  = 95

📤 no-exfiltration

Deny operations suspected of data exfiltration.

deny_when = "data_flow == ExfiltrationSuspected"
scope     = "global"
priority  = 90

🔐 no-privilege-escalation

Deny operations matching known malicious patterns.

deny_when = "pattern == KnownMalicious"
scope     = "global"
priority  = 85

🌐 unknown-destination-review

Escalate outbound requests to unknown destinations for human/LLM review.

escalate_when = "data_flow == Outbound AND target_trust == Unknown"
scope          = "global"
priority       = 50

🧭 goal-drift-detection

Escalate operations that appear unrelated or contradictory to the mission goal.

escalate_when = "goal_relevance == Unrelated OR goal_relevance == Contradictory"
scope          = "global"
priority       = 40

Policies with deny_when block the operation immediately. Policies with escalate_when pause the operation and ask Claude (the LLM judge) to review it. Priority determines evaluation order — the first matching rule wins.
